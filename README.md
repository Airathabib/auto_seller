## АВТОРУСЬ TANK — Лендинг официального дилера

Современный, адаптивный лендинг для официального дилера автомобилей TANK (АВТОРУСЬ). Проект реализован с использованием актуальных технологий веб-разработки, обеспечивает высокую производительность, отличную SEO-оптимизацию и плавный пользовательский опыт на всех устройствах.

# Технологический стек

Фреймворк: Next.js 14 (App Router)
Библиотека: React 18
Язык: TypeScript
Стилизация: SCSS (CSS Modules), глобальные переменные
Иконки: Кастомные SVG-иконки
Шрифты: Локальные шрифты (ttTank)
Структура проекта

# Проект имеет модульную и масштабируемую архитектуру.

app/ — маршрутизация и страницы приложения (главная, страницы моделей).
components/ — все React-компоненты, разделенные на layout (Header, Footer), sections (Hero, CarCatalog и т.д.) и shared/ui (переиспользуемые UI-элементы).
lib/ — хуки (useMediaQuery), константы, типы и настройки шрифтов.
public/ — статические файлы: изображения автомобилей, баннеры, иконки, фавиконки.
styles/ — глобальные стили, сброс (reset) и SCSS-переменные (цвета, отступы, радиусы, шрифты).

# Реализованные страницы

Главная страница (/)
Отображает полный ассортимент. Показывает все доступные модели автомобилей, общие преимущества дилера, блок бронирования и полный каталог с фильтрами по всем моделям.
Страница модели TANK 300 (/cars/tank-300)
Специализированная страница для конкретной модели. При загрузке автоматически показывает промо-баннер, скрывает общие блоки (преимущества и бронирование), отображает только блок с TANK 300 и предустанавливает фильтр модели в каталоге.
Страница модели TANK 500 (/cars/tank-500)
Аналогична странице TANK 300, но настроена для модели TANK 500 с соответствующими изображениями, текстами и фильтрами.

# Основные секции и компоненты

Header (Шапка)
Адаптивная шапка с логотипом (ссылка на главную), адресом, ссылками на сервис и тест-драйв, блоком контактов и кнопкой заказа звонка. На мобильных устройствах превращается в бургер-меню.
Hero (Главный экран)
Полноэкранный баннер с фоновым изображением, заголовком, подзаголовком и CTA-кнопкой, открывающей модальное окно заявки. Поддерживает разные изображения для десктопа и мобильных устройств.
Special Offer (Спецпредложение)
Секция с таймером обратного отсчета, блоком преимуществ дилера и предложением забронировать автомобиль. Поддерживает условный рендеринг внутренних блоков.
Car Offers List (Список предложений)
Детальное представление автомобилей. Включает выбор цвета (ColorPicker) с кастомными тултипами, галерею изображений, список преимуществ (Features) и кнопки действий. Поддерживает режим "reversed" для чередования макетов.
Car Catalog (Каталог)
Интерактивный каталог с фильтрами (Модель, Комплектация, Двигатель). Фильтры реализованы через кастомный компонент Select с выпадающим списком. Карточки автомобилей отображают цену, выгоду по Trade-in и подарки.
Promo Popup (Промо-баннер)
Всплывающее окно с акцией, которое автоматически закрывается через 3 секунды. Показывается только на страницах конкретных моделей. Поддерживает адаптивные изображения.
Contact Section (Контакты)
Секция включает форму заявки на кредит с маской телефона, блок для корпоративных клиентов с фото менеджера и интерактивную Яндекс.Карту с наложенной панелью контактной информации.
Footer (Подвал)
Содержит раскрывающийся дисклеймер, юридическую информацию, контакты, ссылки на сервис и тест-драйв, а также правовые ссылки.
Модальные окна (GetOfferModal)
Кастомное модальное окно для сбора заявок. Включает кастомный Select для выбора модели, поле телефона с маской, чекбокс согласия и блокировку скролла фона.

# Ключевые особенности реализации

Адаптивность: Полная поддержка десктопных, планшетных и мобильных разрешений с использованием хука useMediaQuery и CSS Media Queries.
Условный рендеринг: Компоненты (SpecialOffer, CarOffersList, CarCatalog, CarFilters) принимают пропсы для гибкого управления отображением контента в зависимости от текущей страницы.
Кастомные UI-компоненты: Собственные реализации Select, ContactBlock, ColorPicker и StatusIndicator, полностью стилизованные под дизайн-систему проекта.
Навигация: Глобальный компонент ScrollToTop, который автоматически сбрасывает скролл наверх при переходе между страницами.
Управление состоянием: Использование React Hooks (useState, useEffect, useMemo) для управления фильтрами, модалками и таймерами.

# Установка и запуск

Для запуска проекта локально выполните следующие команды в терминале.
Установите зависимости:
npm install
Запустите сервер разработки:
npm run dev
Откройте браузер и перейдите по адресу:
http://localhost:3000

# Для сборки продакшн-версии используйте команду npm run build, а для запуска собранной версии — npm start.
